**Search strategy for EACH publication (with 2-second delays between API calls):**

Step A — Harvard Dataverse: Use WebFetch to search `https://dataverse.harvard.edu/api/search?q={author_last_name}+{key_title_words}&type=dataset` for each paper. Check if any results match the paper.

Step B — ICPSR: Use WebFetch to search `https://www.icpsr.umich.edu/web/ICPSR/search/studies?q={author_last_name}+{key_title_words}` for each paper.

Step C — Journal article page: For each paper with a DOI, use WebFetch on the DOI URL to load the journal article page. Look for "Supplementary Materials", "Supplementary Data", "Replication Materials", "Data Availability", or "Supporting Information" sections that link to a data repository.

**Important notes:**
- Pace all WebFetch calls with 2-second delays (`sleep 2` between calls) to avoid rate limiting
- For Forthcoming articles, they may not have supplementary materials yet — still check
- Record every search performed and its result (found/not found) in your working notes
- If a deposit is found, verify the URL resolves before adding it
