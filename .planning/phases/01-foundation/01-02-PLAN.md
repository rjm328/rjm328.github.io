<task type="auto">
  <name>Task 2: Wire real content and deploy to GitHub Pages</name>
  <files>_quarto.yml, index.qmd, contact.qmd, images/profile.jpg</files>
  <action>
Using the content provided in Task 1:

1. **Optimize the profile photo:**
   - Copy the user's photo to `images/profile.jpg`
   - Resize to max 600px on longest side: `sips -Z 600 images/profile.jpg`
   - Check file size: `ls -la images/profile.jpg`
   - If over 150 KB, reduce quality: `sips -s formatOptions 80 images/profile.jpg`
   - Verify final size is <=150 KB

2. **Update `_quarto.yml`:**
   - Replace `title: "Researcher Name"` with the user's real name (from Task 1 item 1)

3. **Update `index.qmd`:**
   - Replace `title:` with the user's full name (item 1)
   - Replace `subtitle:` with the user's affiliation (item 2)
   - Replace `href: mailto:placeholder@university.edu` with real email (item 5)
   - Update Google Scholar link href (item 7) — if "skip", remove that links entry entirely
   - Update ORCID link href (item 8) — if "skip", remove that links entry entirely
   - Replace the bio placeholder paragraph with the real bio text (item 3)
   - Replace the research interests placeholder with the real research interests text (item 4)

4. **Update `contact.qmd`:**
   - Replace placeholder email with the real email address (item 5)

5. **Verify locally:**
   - Run `bash scripts/check-render.sh` to confirm build still passes
   - Run `quarto preview` briefly to visually confirm real content appears

6. **Initialize git and create GitHub repo:**
   ```bash
   git init
   git add _quarto.yml custom.scss index.qmd publications.qmd cv.qmd contact.qmd .gitignore .nojekyll images/ scripts/
   git add .planning/
   git commit -m "Initial Quarto site scaffold with warm neutral theme and content"
   gh repo create <repo-name> --public --source=. --remote=origin --push
   ```
   Use the repo name from Task 1 item 10 and confirm the GitHub username from item 9.

7. **Deploy to GitHub Pages:**
   ```bash
   quarto publish gh-pages --no-browser
   ```
   This creates the `gh-pages` branch, renders the site, and pushes. The `--no-browser` flag prevents auto-opening.

8. **Verify HTTPS:**
   - Wait 60 seconds for GitHub Pages to provision
   - Check the site is accessible: `curl -sI https://<username>.github.io/<repo-name>/ | head -5`
   - Confirm HTTPS (200 status code, no redirect to HTTP)
   - If HTTPS is not enforced, note it for the checkpoint in Task 3

Note on INFR-02 (custom domain): Per user decision in CONTEXT.md, custom domain is deferred. The site deploys to the default GitHub Pages URL. INFR-02 will be fully satisfied when the user acquires a domain (out-of-band task). Phase 1 success criterion 1 is satisfied by the GitHub Pages URL with HTTPS.
