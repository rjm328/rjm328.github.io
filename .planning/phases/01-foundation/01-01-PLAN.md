  <files>_quarto.yml, custom.scss, .gitignore, .nojekyll, scripts/check-render.sh</files>
  <action>
1. Verify Quarto is installed: run `quarto --version`. If not installed, run `brew install --cask quarto`.

2. Scaffold the Quarto website project in the current directory (which is empty except for `.planning/`):
   ```
   quarto create project website .
   ```
   This creates `_quarto.yml`, `index.qmd`, `styles.css`, and `_quarto/`. Delete the generated `styles.css` (we replace it with `custom.scss`). Delete the generated `about.qmd` if present (we use `index.qmd` as the combined home/about page).

3. Replace the generated `_quarto.yml` with this exact content:
   ```yaml
   project:
     type: website

   website:
     title: "Researcher Name"
     navbar:
       background: "#FAF8F5"
       foreground: "#3D3632"
       left:
         - text: "Home"
           href: index.qmd
         - text: "Publications"
           href: publications.qmd
         - text: "CV"
           href: cv.qmd
         - text: "Contact"
           href: contact.qmd

   format:
     html:
       theme:
         - cosmo
         - custom.scss
       toc: false
       page-layout: full
   ```
   Note: The title "Researcher Name" is a placeholder — it will be replaced with the user's real name in Plan 02.

4. Create `custom.scss` with these exact contents:
   ```scss
   /*-- scss:defaults --*/

   @import url('https://fonts.googleapis.com/css2?family=Source+Serif+4:ital,opsz,wght@0,8..60,300..900;1,8..60,300..900&family=DM+Sans:ital,opsz,wght@0,9..40,300..900;1,9..40,300..900&display=swap');

   /* Warm neutrals palette — locked by user decision */
   $body-bg:       #FAF8F5;
   $body-color:    #3D3632;
   $link-color:    #8B5E3C;
   $navbar-bg:     #FAF8F5;
   $navbar-fg:     #3D3632;
   $navbar-hl:     #C4813D;
   $footer-bg:     #EDE9E4;

   /* Typography — Source Serif 4 for body, DM Sans for headings and UI */
   $font-family-sans-serif: "DM Sans", system-ui, -apple-system, sans-serif;
   $font-family-base:       "Source Serif 4", Georgia, "Times New Roman", serif;
   $headings-font-family:   "DM Sans", system-ui, sans-serif;
   $headings-font-weight:   600;

   /*-- scss:rules --*/

   body {
     -webkit-font-smoothing: antialiased;
     -moz-osx-font-smoothing: grayscale;
   }

   /* Narrow centered content column — Distill.pub-inspired */
   .quarto-container {
     max-width: 720px;
   }

   /* Trestles mobile fix — prevent photo touching navbar */
   @media (max-width: 768px) {
     .about-image {
       margin-top: 1.5rem;
     }
   }
   ```

   Both `/*-- scss:defaults --*/` and `/*-- scss:rules --*/` region decorators are REQUIRED. Quarto silently ignores SCSS without them.

5. Create `.gitignore`:
   ```
   /_site/
   /.quarto/
   .DS_Store
   ```

6. Create `.nojekyll` as an empty file (prevents GitHub Pages from running Jekyll on output).

7. Create `scripts/check-render.sh` (Wave 0 smoke test):
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail
   echo "Running Quarto render..."
   quarto render --no-cache 2>&1 | tee /tmp/quarto-render.log
   if grep -qi "error" /tmp/quarto-render.log; then
     echo "FAIL: Quarto render produced errors"
     exit 1
   fi
   echo "PASS: Quarto render completed without errors"
   ```
   Make it executable: `chmod +x scripts/check-render.sh`
  </action>
  <verify>
    <automated>bash scripts/check-render.sh</automated>
  </verify>
  <done>Quarto renders without errors. `_quarto.yml` configures website type with 4-item navbar. `custom.scss` applies warm neutrals palette and Source Serif 4 / DM Sans typography with both region decorators present. `.gitignore` excludes `_site/` and `.quarto/`. `.nojekyll` exists.</done>
