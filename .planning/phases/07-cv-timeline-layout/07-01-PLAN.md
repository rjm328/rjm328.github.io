**cv.qmd changes:**

Restructure every CV entry into a two-column timeline pattern using Quarto fenced divs. The new markup pattern for each entry is:

```
::: {.cv-timeline}
::: {.cv-timeline-date}
2017 -- present
:::
::: {.cv-timeline-content}
**Associate Professor, Schar School**
:::
:::
```

Apply this transformation to ALL sections:

1. **Education** -- Extract the date portion from each title. For the Ph.D. entry, the date is "July 2011"; for the B.A. entry, the date is "2004". Move the degree name (bold), institution, and any details (dissertation, committee) to the content column.

2. **Academic Positions** -- These already have separate `.cv-entry-meta` dates. Move the date to `.cv-timeline-date`. The institution header lines ("**George Mason University** -- Fairfax, Virginia" and "**University of Michigan** -- Ann Arbor, Michigan") should appear as their own full-width subheadings ABOVE the timeline entries for that institution. Use a `{.cv-institution}` div for these.

3. **Publications** -- Keep the summary paragraph and "View full publications list" button OUTSIDE the timeline (they are not timeline entries). For the 3 Book Reviews entries, extract any parenthetical date or publication year to the date column. Book reviews have dates like "(Spring 2025)", "(April 2020)", "(April 2016)" -- extract just the year portion (2025, 2020, 2016) for the date column and keep the full citation text in the content column.

4. **Awards & Fellowships** -- This section has three subsections (Research Awards, Teaching Awards, Grants & Fellowships). Keep the h3 subsection headers. For each entry, extract the year(s) to the date column. Years appear at end of entries (e.g., "2015", "2023", "2022", "2013 -- 2015", "2012", "2009", "2008", "2005", "2004 -- 2010"). If an entry has a date range like "Oct 1, 2016 -- Sep 30, 2019", simplify to "2016 -- 2019" in the date column.

Keep the existing `.cv-section` fenced divs wrapping each section (they provide the border-top separator). The timeline entries go INSIDE the `.cv-section` divs.

Keep the PDF download button div at the very top exactly as-is.

Keep the `## Section` h2 headers exactly as-is (they already have gold underlines from Phase 5).

**custom.scss changes:**

Add a new comment block after the existing `.cv-entry-meta` rules (after line 105). Keep ALL existing `.cv-section`, `.cv-entry`, `.cv-entry-title`, `.cv-entry-meta` rules in place (they may still be used outside the timeline or as fallbacks). Add:

```scss
/* CV timeline layout — Phase 7 CVFX-01 */
.cv-timeline {
  display: grid;
  grid-template-columns: 120px 1fr;
  gap: 0 1.25rem;
  margin-bottom: 0.75rem;
  align-items: baseline;
}

.cv-timeline-date {
  font-family: "DM Sans", system-ui, sans-serif;
  font-size: 0.85rem;
  color: #6B5E55;
  white-space: nowrap;
  padding-top: 0.15rem;
}

.cv-timeline-content {
  /* Inherits body font (Source Serif 4) */
}

.cv-timeline-content strong {
  font-family: "DM Sans", system-ui, sans-serif;
}

/* Institution subheading spans full width above its entries */
.cv-institution {
  font-weight: 600;
  font-family: "DM Sans", system-ui, sans-serif;
  margin-bottom: 0.5rem;
  margin-top: 0.75rem;
}

.cv-institution:first-child {
  margin-top: 0;
}

/* Phase 7 CVFX-02 — enhanced section separators */
.cv-section {
  margin-top: 2rem;
  padding-top: 1.25rem;
  border-top: 2px solid #EDE9E4;
}

/* Responsive: stack on mobile */
@media (max-width: 576px) {
  .cv-timeline {
    grid-template-columns: 1fr;
    gap: 0;
  }

  .cv-timeline-date {
    margin-bottom: 0.15rem;
  }
}
```

Note: The `.cv-section` rule above replaces the existing one (lines 87-91 of current custom.scss). Update those values in-place rather than duplicating the rule. Change border-top from `1px solid` to `2px solid` for stronger separation per CVFX-02, and increase margin-top from 1.5rem to 2rem for clearer section breaks while staying consistent with the tighter Phase 5 density (not going back to the pre-Phase-5 2.5rem).
